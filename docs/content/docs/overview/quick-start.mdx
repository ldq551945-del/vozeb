# 快速开始

如果你只是想先把项目跑起来，优先使用发布镜像或启动 `web/` 前端。

## Docker 启动

低配服务器推荐直接拉取 VOZEB 发布镜像，服务器只负责启动容器，不现场执行 `next build`：

```bash
git clone git@github.com:csyqlz/vozeb.git
cd vozeb
docker compose pull
docker compose up -d
```

启动后访问：

```text
https://你的域名
```

默认 Compose 使用 `VOZEB_BIND_HOST=0.0.0.0`，`http://服务器IP:3000` 直连和域名反向代理都可用。域名部署时，请把域名解析到服务器，并通过 Nginx、Caddy 或宝塔反向代理到 `http://127.0.0.1:3000`。建议在 `.env` 中填写：

```env
NEXT_PUBLIC_SITE_URL=https://你的域名
VOZEB_COOKIE_SECURE=true
```

反向代理需要保留 `Host`、`X-Forwarded-Host` 和 `X-Forwarded-Proto` 请求头。

发布镜像版和本地构建版都默认使用：

```yaml
ports:
  - "${VOZEB_BIND_HOST:-0.0.0.0}:3000:3000"
```

默认值 `0.0.0.0` 会让 IP 直连和域名反代都可用。如果只想允许域名反代访问，不想暴露 `服务器IP:3000`，在 `.env` 中设置：

```env
VOZEB_BIND_HOST=127.0.0.1
```

如果拉取 `ghcr.io/csyqlz/vozeb:latest` 提示 `denied`，说明 GitHub Packages 里的镜像还没有设为 Public，或者服务器没有登录 GHCR。公开部署请进入 GitHub Packages 的 `vozeb` 页面，把包可见性改为 Public；私有部署请先执行 `docker login ghcr.io`。

## 本地启动

```bash
git clone git@github.com:csyqlz/vozeb.git
cd vozeb/web
pnpm install
pnpm run dev
```

启动后访问：

```text
http://localhost:3000
```

## Vercel 部署

在 Vercel 中导入仓库即可，根目录的 `vercel.json` 会构建 `web/`。模型渠道、默认模型、WebDAV 和真实上游连接信息统一由管理员后台配置；前端只使用站内系统代理地址，不再接收用户自定义 Base URL 或 API Key。图片、视频和文本续取类生成会通过服务端任务接口保存任务状态，不需要额外部署 AI 转发服务。

## 首次使用建议

- 第一次注册的账号会自动成为管理员，这个账号用于初始化网站和进入后台管理。
- 管理员登录后进入 `管理员后台`，先配置网站标题、Logo、SEO、首页提示词展示、注册策略、邮箱服务、模型渠道、默认模型、用户积分和公共提示词库。
- `http://服务器IP:3000` 直连部署会使用 HTTP 登录 Cookie；HTTPS 反向代理需要保留 `X-Forwarded-Proto: https`，特殊环境可用 `VOZEB_COOKIE_SECURE=true/false` 强制指定。
- 管理员公共提示词库会自动采集原作者接入过的远程提示词源，并只保留可访问的远程图片 URL。
- 首页提示词展示可以随机读取公共提示词库，也可以在后台自定义展示内容；随机模式会懒加载，适合低配服务器保留首屏速度。
- 如果需要跨设备同步画布、素材和生成记录，请由管理员在后台系统设置中配置 WebDAV。

## 数据说明

- 画布项目和“我的素材”默认主要保存在浏览器本地；管理员开启 WebDAV 后，登录用户可通过系统代理同步画布、素材、生成记录和本地媒体缓存。
- 账号、后台设置、签到记录、公共提示词和生成日志会写入服务端 `.data` 目录。
- 生成结果服务器副本保存在 `.data/generation-assets`，是否保存图片/视频副本由管理员后台的下载保存开关分别控制。
- Docker Compose 默认挂载 `vozeb-data` 数据卷，正常执行 `docker compose pull && docker compose up -d` 更新镜像不会删除已有用户。
- 不要执行 `docker compose down -v`，否则会删除数据卷。
