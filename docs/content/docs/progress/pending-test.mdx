# 待测试

- 手机端顶部操作区：应显示紧凑积分入口，点击可查看积分记录；版本和 GitHub 在窄屏隐藏，不挤压品牌与常用操作。
- 签到按钮：已签到状态不应切换特殊背景色，应使用与其他顶部按钮一致的浅色/深色底色，只通过文字显示状态。
- 顶部右侧操作区：签到、设置、主题、版本、GitHub 和账号入口应使用统一按钮 UI，间距舒适；签到不再出现突兀亮蓝色。
- 画布顶部动作区：积分、设置、主题、版本、GitHub、账号、快捷键和 Agent 入口应在桌面端尺寸统一、间距均匀，手机端只保留必要入口且不溢出。
- 画布图片组：右上角数量按钮不应使用固定蓝色，数字、箭头、背景和边框应跟随浅色/深色画布主题；手机端顶部 Agent 按钮高度应与设置、主题按钮一致。
- 管理员后台生成日志：手机端日期范围控件应整行对齐筛选区，开始/结束日期均分显示，不再悬在中间或出现松散空白。
- 顶部签到入口：普通页面未签到显示“签到”按钮，签到成功后变为“已签到”，画布顶部不显示签到按钮。
- 生图工作台：点击左侧生成记录只切换提示词、配置和右侧结果，不应自动把该记录参考图插入当前参考图区。
- 画布输入框：聚焦输入、选择文本、插入 `@节点` 引用和继续编辑时，光标应稳定显示；失焦后引用标签仍保持高亮。
- 本地 standalone 预览：执行 `pnpm start:standalone` 后，首页 CSS 和 logo 应返回 200，页面不应退化成裸样式。
- 画布反推提示词：生成中刷新页面后，文本节点应继续轮询服务端文本任务并回填结果；失败后点击重试应继续读取原始图片引用。
- 管理员后台生成日志：桌面端日期范围放在第一行右侧时，中间“至”和日历图标不应出现过大空白；手机端筛选区不横向溢出。
- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 管理员后台：优化后台首页排版，改为侧边栏切换概览、系统设置、用户管理和公共提示词库。
- 品牌与版本：项目展示名更新为 VOZEB，右上角 GitHub 和检查更新地址切换到 csyqlz/vozeb。
- 邮箱注册：后台开启邮箱注册后，前台注册页显示邮箱输入框；SMTP 配置支持 QQ、网易和自定义邮箱，并可通过“测试邮箱”验证发信。
- 管理员提示词库：自动采集原作者接入的远程提示词源，确认封面图为真实远程图片 URL，且不再出现 img.shields.io 徽章图。
- 全局通知：签到成功等 Ant Design 消息下移到顶部导航下方，并统一浅色/深色样式。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
